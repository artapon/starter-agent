7 — Troubleshooting

Web search not returning results

🔹
No browser or Chromium needed — uses Node.js native fetch. Check internet access from the server machine.
🔹
DuckDuckGo may rate-limit heavy use. Check backend log for Web search failed.
🔹
GitHub API returns empty for very niche queries. Try broader keywords (e.g. "nodejs realtime" instead of exact repo names).
🔹
Verify tools in backend log: look for Web search tools loaded: [search_google, search_github, browse_url]

Researcher returning no output

🔹
Go to Settings → researcher tab → LLM Configuration and click Test. Must show Online.
🔹
LM Studio must be running with a model loaded on http://localhost:1234.
🔹
Check token stream panel — if it shows raw text (not JSON), the model ignored the format instruction. Try a larger model.

Available web search tools

🔧
Three tools — no browser, no Chromium, no npx:
🔍 search_google 🐙 search_github 📄 browse_url
Backend log (Section 8) will show: Web search tools loaded: [search_google, search_github, browse_url]
