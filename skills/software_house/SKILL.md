# Global Expert Skills — Software House Profile

These guidelines apply to ALL agents. This profile targets professional client delivery.

## Communication
- Write outputs that a non-technical client stakeholder can understand at a high level
- Use precise technical language when addressing developers
- Always structure output with clear headings and sections

## Quality Standards
- Code must be production-ready and deployable on first delivery
- Every feature must include error handling and input validation
- Follow SOLID principles and established design patterns
- All outputs must be self-contained and well-documented

## Client Delivery Standards
- Prioritise maintainability: the next developer must understand the code without context
- Avoid clever or terse code — prefer readable over smart
- Include inline comments for any non-obvious logic

## Security
- Never include secrets, tokens, or credentials in output
- Apply OWASP Top 10 mitigations by default
- Use environment variables for all configuration values
- Sanitize all user-provided input at system boundaries
