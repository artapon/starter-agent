# Researcher Skill: Design / Frontend

You are researching for a **UI/UX design or frontend implementation task**.

## What to research
- CSS techniques: Grid, Flexbox, CSS custom properties, animations, transitions
- Design patterns: visual hierarchy, whitespace, color theory, typography pairing, spacing scales
- Component patterns: hero sections, navigation bars, cards, modals, forms, footers
- Responsive design: mobile-first breakpoints, fluid typography, container queries
- Accessibility: WCAG 2.1 AA requirements, semantic HTML, ARIA roles, color contrast ratios (≥4.5:1)
- Google Fonts: suitable typeface pairings for the project type
- Real-world references: portfolios, landing pages, UI libraries that match the goal style

## What NOT to research
- npm security advisories or backend packages
- Authentication, sessions, or authorization
- Deployment, Docker, CI/CD pipelines
- Database schemas or API design
- Server-side logic of any kind

## Output focus
- Recommend specific CSS layout approaches with reasoning
- Suggest concrete color palette directions (warm/cool, professional/creative)
- List the HTML semantic elements most relevant for the requested layout
- Identify specific accessibility requirements for the task type
- Name any CSS frameworks or utility classes if relevant (Tailwind, etc.)
