# Reviewer Skill: Design / Frontend

Evaluate **only** design and frontend quality. This is a visual output task.

## Do NOT penalise for missing
- Version control (git, commits, branches)
- README or documentation files
- Backend security hardening
- Deployment or CI/CD configuration
- npm install instructions

These are process artifacts irrelevant to HTML/CSS output quality.

## Evaluation criteria

1. **Visual Quality** — Polished and professional: appropriate whitespace, clear visual hierarchy, balanced layout, consistent spacing
2. **Real Content** — All sections have meaningful, realistic content. No "Lorem ipsum", no "Your name here", no "Add bio here"
3. **CSS Craft** — CSS custom properties defined and used, logical class names, no redundant rules, responsive breakpoints present
4. **Semantic HTML** — Correct heading hierarchy (h1→h2→h3), landmark elements (nav, main, section, footer), descriptive alt text on images
5. **Responsiveness** — Works on mobile (320px) and desktop (1280px+), no horizontal scroll on mobile
6. **Accessibility** — Sufficient color contrast (WCAG AA ≥ 4.5:1), keyboard-navigable interactive elements, focus styles visible

## Score guide
- 9–10: Genuinely beautiful, complete, professional. Real content. Responsive. Accessible. Would pass a design review.
- 7–8: Good design, complete sections, only minor visual or CSS issues.
- 5–6: Recognisable structure but generic/incomplete content, basic styling only, not responsive.
- 3–4: Placeholder content throughout, minimal CSS, not polished.
- 1–2: Broken layout, no real content, or completely off-task.

## Feedback rules
- Every suggestion must describe a concrete change to the HTML or CSS
- Never suggest: git commits, README, deployment setup, npm install
